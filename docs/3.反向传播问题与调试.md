# 反向传播问题与调试指南

## 1. 梯度消失问题（Vanishing Gradient）

**问题表现**：深层网络中，梯度越来越小，导致前面层几乎不更新

**数学原因**：
```python
# 假设使用sigmoid激活函数
sigmoid(x) = 1/(1+e^(-x))
sigmoid'(x) = sigmoid(x)*(1-sigmoid(x)) ≤ 0.25

# 经过多层传播后：
∂loss/∂w₁ = ∂loss/∂y * f'(zₙ) * f'(zₙ₋₁) * ... * f'(z₁) * x
          ≤ 0.25^n * 常数  # 指数级衰减！
```

**解决方案**：
- 使用ReLU激活函数：`f'(x) = 1 (当x>0)`
- 残差连接（ResNet）
- 批量归一化

## 2. 梯度爆炸问题（Exploding Gradient）

**问题表现**：梯度变得极大，导致参数更新过大，模型不稳定

**数学原因**：
```python
# 当权重w > 1时
∂loss/∂w₁ = ∂loss/∂y * wₙ * wₙ₋₁ * ... * w₁ * x
          = 常数 * w^n  # 指数级增长！
```

**解决方案**：
- 梯度裁剪（Gradient Clipping）
- 权重初始化技巧
- 学习率调整

## 3. 数值不稳定问题

**常见情况**：
```python
# 1. 除零错误
loss = -log(y_pred)  # 当y_pred=0时，log(0)=∞

# 2. 数值溢出
exp(1000) = ∞  # 在softmax等函数中常见

# 3. 精度损失
0.1 + 0.2 = 0.30000000000000004  # 浮点数精度问题
```

## 4. 局部最优和鞍点问题

**数学表现**：
```
∂loss/∂w = 0  # 梯度为零，但可能不是全局最优
```

**鞍点示例**：
```
loss = x² - y²
∂loss/∂x = 2x = 0 在x=0
∂loss/∂y = -2y = 0 在y=0
但(0,0)是鞍点，不是最小值！
```

## 5. 实际代码中的问题检测

### 梯度检查
```python
# 手动计算梯度 vs 自动求导
def manual_gradient(w, x, y_true):
    y_pred = w * x
    loss = (y_pred - y_true)**2
    return 2 * (y_pred - y_true) * x

# 比较差异
auto_grad = torch.autograd.grad(loss, w)[0]
manual_grad = manual_gradient(w, x, y_true)
diff = torch.abs(auto_grad - manual_grad)
print(f"梯度差异: {diff}")
```

### 梯度监控
```python
# 检查梯度是否正常
for name, param in model.named_parameters():
    if param.grad is not None:
        grad_mean = param.grad.mean().item()
        grad_std = param.grad.std().item()
        print(f"{name}: mean={grad_mean:.6f}, std={grad_std:.6f}")
        
        # 问题检测
        if torch.isnan(param.grad).any():
            print(f"⚠️ {name} 出现NaN梯度！")
        if torch.isinf(param.grad).any():
            print(f"⚠️ {name} 出现无穷大梯度！")
```

## 6. 常见问题症状和调试方法

### 症状1：损失不下降
```python
# 可能原因：梯度消失
if torch.all(param.grad == 0):
    print("梯度为零！检查激活函数或网络结构")
```

### 症状2：损失变为NaN
```python
# 可能原因：梯度爆炸或数值不稳定
if torch.isnan(loss):
    print("损失为NaN！检查学习率、梯度裁剪")
```

### 症状3：训练不稳定
```python
# 监控梯度范数
total_norm = 0
for p in model.parameters():
    if p.grad is not None:
        param_norm = p.grad.data.norm(2)
        total_norm += param_norm.item() ** 2
total_norm = total_norm ** 0.5
print(f"梯度总范数: {total_norm}")

if total_norm > 1000:  # 阈值
    print("梯度爆炸！需要梯度裁剪")
```

## 8. 完整的训练循环调试示例

```python
import torch
import torch.nn as nn

def train_with_debug(model, train_loader, criterion, optimizer, num_epochs):
    model.train()
    
    for epoch in range(num_epochs):
        running_loss = 0.0
        
        for i, (inputs, labels) in enumerate(train_loader):
            # 前向传播
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            
            # 检查损失
            if torch.isnan(loss):
                print(f"Epoch {epoch}, Batch {i}: 损失为NaN！")
                break
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            
            # 梯度监控
            total_norm = 0
            for name, param in model.named_parameters():
                if param.grad is not None:
                    param_norm = param.grad.data.norm(2)
                    total_norm += param_norm.item() ** 2
                    
                    # 检查梯度问题
                    if torch.isnan(param.grad).any():
                        print(f"⚠️ {name}: NaN梯度")
                    if torch.isinf(param.grad).any():
                        print(f"⚠️ {name}: 无穷大梯度")
            
            total_norm = total_norm ** 0.5
            
            # 梯度裁剪（如果需要）
            if total_norm > 1000:
                print(f"梯度爆炸！范数: {total_norm:.2f}")
                torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            
            # 参数更新
            optimizer.step()
            
            running_loss += loss.item()
            
            if i % 100 == 99:
                print(f'Epoch {epoch}, Batch {i+1}, Loss: {running_loss/100:.4f}')
                running_loss = 0.0
```
