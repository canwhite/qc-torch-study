# advance/4_save_load_model.py 完整链条分析：

1. 模型定义链条

模型类定义 → 网络层构建 → 正向传播路径

- TheModelClass.**init**(): 定义网络结构（conv1, pool, conv2, fc1, fc2, fc3）
- forward(): 定义数据流动路径（输入 → 卷积 → 池化 → 全连接 → 输出）

```
  forward()方法的数据流动路径：

  def forward(self, x):
      # 输入x: [batch_size, 3, 32, 32] 假设是32x32的RGB图像

  1. 第一层：卷积层1 + ReLU激活 + 池化层

  x = self.pool(F.relu(self.conv1(x)))

  conv1的作用：
  - 输入：[batch_size, 3, 32, 32] (3个颜色通道)
  - 输出：[batch_size, 6, 28, 28] (6个特征图)
  - 做什么：提取低级视觉特征
    - 边缘检测
    - 纹理识别
    - 颜色变化检测
  - 数学：output = conv2d(input, weight) + bias

  ReLU激活：
  - 做什么：引入非线性，过滤负值
  - 数学：f(x) = max(0, x)
  - 作用：解决梯度消失，计算简单

  池化层：
  - 输入：[batch_size, 6, 28, 28]
  - 输出：[batch_size, 6, 14, 14]
  - 做什么：下采样，保留最显著特征
  - 作用：减少计算量，增强平移不变性

  2. 第二层：卷积层2 + ReLU激活 + 池化层

  x = self.pool(F.relu(self.conv2(x)))

  conv2的作用：
  - 输入：[batch_size, 6, 14, 14]
  - 输出：[batch_size, 16, 10, 10]
  - 做什么：提取更复杂的特征组合
    - 形状组合
    - 模式识别
    - 高级特征抽象

  池化层：
  - 输入：[batch_size, 16, 10, 10]
  - 输出：[batch_size, 16, 5, 5]

  3. 展平操作

  x = x.view(-1, 16 * 5 * 5)
  - 输入：[batch_size, 16, 5, 5]
  - 输出：[batch_size, 400] (16×5×5=400)
  - 做什么：将空间特征转换为一维向量
  - 作用：为全连接层准备输入

  4. 全连接层1 + ReLU激活

  x = F.relu(self.fc1(x))
  - 输入：[batch_size, 400]
  - 输出：[batch_size, 120]
  - 做什么：特征组合和抽象
  - 作用：学习特征之间的复杂关系

  5. 全连接层2 + ReLU激活

  x = F.relu(self.fc2(x))
  - 输入：[batch_size, 120]
  - 输出：[batch_size, 84]
  - 做什么：进一步特征抽象
  - 作用：为最终分类做准备

  6. 输出层

  x = self.fc3(x)
  - 输入：[batch_size, 84]
  - 输出：[batch_size, 10] (假设10分类)
  - 做什么：生成最终的分类得分
  - 作用：输出每个类别的概率（logits）

  完整的数据变换链条：

  原始图像 → 低级特征 → 高级特征 → 展平向量 → 特征组合 → 分类得分
  [b,3,32,32] → [b,6,14,14] → [b,16,5,5] → [b,400] → [b,120] → [b,84] → [b,10]

  每层的"职责"总结：

  1. 卷积层：特征提取专家
  2. 池化层：信息压缩专家
  3. 全连接层：关系学习专家
  4. 激活函数：非线性引入专家
  5. 输出层：决策专家

  这样层层递进，从像素到特征，再到最终分类决策！
```

1. 训练循环链条

数据加载 → 正向传播 → 损失计算 → 反向传播 → 参数更新

#### 完整训练链条

inputs, labels = data # 1. 数据输入
optimizer.zero_grad() # 2. 梯度清零
outputs = model(inputs) # 3. 正向传播
loss = criterion(outputs, labels) # 4. 损失计算  
 loss.backward() # 5. 反向传播（自动微分）
optimizer.step() # 6. 参数更新

3. 反向传播的数学链条

loss → fc3 → fc2 → fc1 → 展平 → conv2 → conv1 → 输入
使用链式法则：
∂loss/∂w*fc3 = ∂loss/∂outputs * ∂outputs/∂w*fc3
∂loss/∂w_fc2 = ∂loss/∂fc3_output * ∂fc3_output/∂fc2_output \* ∂fc2_output/∂w_fc2
...

4. 模型保存链条

训练完成 → 获取 state_dict → 序列化保存 → 磁盘文件

#### 保存参数

torch.save(model.state_dict(), save_path) # 只保存参数

#### 保存整个模型

torch.save(model, save_model_path) # 保存模型+参数

5. 模型加载链条

磁盘文件 → 反序列化 → 加载 state_dict → 模型恢复

#### 加载参数

model.load_state_dict(torch.load(save_path))
model.eval() # 切换到推理模式

##### 加载整个模型

model = torch.load(save_model_path)
model.eval()

6. 检查点保存链条（用于继续训练）

训练状态 → 打包所有信息 → 序列化保存
torch.save({
'epoch': epoch,
'model_state_dict': model.state_dict(),
'optimizer_state_dict': optimizer.state_dict(),
'loss': loss,
}, checkpoint_path)

7. 设备管理链条

模型/数据 → 设备转移 → 训练/推理 → 保存/加载

#### CPU ↔ GPU 设备链条

model.to(device)
inputs = inputs.to(device)

关键链条总结：

1. 训练链：数据 → 模型 → 损失 → 梯度 → 更新
2. 保存链：模型状态 → 序列化 → 文件存储
3. 加载链：文件 → 反序列化 → 模型恢复
4. 反向传播链：损失 → 输出层 → 隐藏层 → 输入层

每个链条都体现了深度学习的核心原理：数据流动、梯度计算、状态管理。
